1. WEBSITE PURPOSE
================================================================================

Ironix Hardware Shop is a comprehensive e-commerce platform designed to provide
an online shopping experience for hardware products and tools. The website serves
as a digital storefront where customers can browse, search, and purchase various
hardware items with ease.

PRIMARY OBJECTIVES:
-------------------
- Provide an intuitive online shopping experience for hardware products
- Enable customers to browse products, add items to cart, and complete purchases
- Offer administrative tools for managing products, orders, customers, and inventory
- Support supplier management and order tracking
- Deliver a modern, responsive user interface accessible on all devices

TARGET AUDIENCE:
---------------
- Individual consumers looking to purchase hardware tools and equipment
- Professional contractors and builders
- DIY enthusiasts
- Business customers requiring bulk hardware supplies

BUSINESS MODEL:
--------------
- B2C (Business-to-Consumer) e-commerce platform
- Direct product sales with online ordering
- Free shipping on all orders
- Free returns up to 90 days
- Secure checkout and payment processing

DATABASE STRUCTURE:
-----------------

1. Products Table
   - id (INT, Primary Key, Auto Increment)
   - name (VARCHAR 255)
   - description (TEXT)
   - price (DECIMAL 10,2)
   - image_url (VARCHAR 500)
   - stock_quantity (INT)
   - supplier_id (INT, Foreign Key)
   - created_at, updated_at (TIMESTAMP)

2. Users Table
   - id (INT, Primary Key, Auto Increment)
   - name (VARCHAR 255)
   - email (VARCHAR 255, Unique)
   - phone (VARCHAR 20)
   - address (TEXT)
   - created_at, updated_at (TIMESTAMP)

3. Cart Table
   - id (INT, Primary Key, Auto Increment)
   - product_id (INT, Foreign Key)
   - name (VARCHAR 255)
   - price (DECIMAL 10,2)
   - quantity (INT)
   - created_at, updated_at (TIMESTAMP)

4. Orders Table
   - id (INT, Primary Key, Auto Increment)
   - user_id (INT, Foreign Key)
   - order_number (VARCHAR 50, Unique)
   - total_amount (DECIMAL 10,2)
   - status (VARCHAR 50)
   - shipping_address (TEXT)
   - order_date (TIMESTAMP)
   - created_at, updated_at (TIMESTAMP)

5. Order Items Table
   - id (INT, Primary Key, Auto Increment)
   - order_id (INT, Foreign Key)
   - product_id (INT, Foreign Key)
   - product_name (VARCHAR 255)
   - quantity (INT)
   - unit_price (DECIMAL 10,2)
   - subtotal (DECIMAL 10,2)
   - created_at (TIMESTAMP)

6. Suppliers Table
   - id (INT, Primary Key, Auto Increment)
   - name (VARCHAR 255)
   - email (VARCHAR 255)
   - phone (VARCHAR 20)
   - address (TEXT)
   - company_name (VARCHAR 255)
   - contact_person (VARCHAR 255)
   - created_at, updated_at (TIMESTAMP)

7. Admin Table
   - id (VARCHAR 50, Primary Key)
   - name (VARCHAR 255)
   - email (VARCHAR 255)
   - password (VARCHAR 255)
   - gender (VARCHAR 10)
   - created_at, updated_at (TIMESTAMP)

8. Blog Table
   - id (INT, Primary Key, Auto Increment)
   - title (VARCHAR 255)
   - content (TEXT)
   - name (VARCHAR 255)
   - description (TEXT)
   - price (DECIMAL 10,2)
   - image_url (VARCHAR 500)
   - author (VARCHAR 100)
   - published_at (TIMESTAMP)
   - created_at, updated_at (TIMESTAMP)

API ENDPOINTS:

6. USER FLOW
================================================================================

CUSTOMER JOURNEY:
----------------

FLOW 1: BROWSING AND PURCHASING
--------------------------------
1. User visits homepage (index.php)
   - Sees promotional banner
   - Views featured products in grid
   - Can use search bar

2. User browses products
   - Clicks "Products" in navigation
   - Views all products page
   - Sees product cards with images and prices

3. User searches for product
   - Types in search bar
   - Presses Enter
   - Redirected to search_results.php
   - Views filtered products

4. User adds product to cart
   - Clicks add to cart button (icon or full button)
   - Product added via AJAX
   - Redirected to cart.php
   - Sees product in cart

5. User views cart
   - Sees all cart items
   - Views order summary with total
   - Can remove items
   - Can continue shopping

6. User proceeds to checkout
   - Clicks "Checkout" button
   - Redirected to checkout.php
   - Reviews order details
   - Sees customer information
   - Views final total

7. User completes purchase
   - Clicks "Buy Now" button
   - Order processed via process_order.php
   - Order created in database
   - Redirected to order_success.php
   - Sees confirmation message

FLOW 2: USER REGISTRATION AND LOGIN
------------------------------------
1. User clicks user icon
   - Dropdown menu appears
   - Options: Login, Register

2. User chooses registration
   - Clicks "Register"
   - Redirected to registration.php
   - Fills registration form:
     * Name
     * Email
     * Phone
     * Address
   - Submits form

3. Registration processing
   - Email uniqueness checked
   - If email exists: Error message shown
   - If email new: User created in database
   - Session created
   - Auto-login performed
   - Redirected to homepage

4. User chooses login
   - Clicks "Login"
   - Redirected to login.html/login.php
   - Enters credentials
   - Session created
   - Redirected to homepage

FLOW 3: EXPLORING WEBSITE PAGES
--------------------------------
1. User navigates menu
   - Clicks "Features" → features.php
   - Clicks "Services" → services.php
   - Clicks "Blog" → blog.php
   - Clicks "Contact" → contact.php
   - Clicks "Support" → support.php

2. User can return to shopping
   - Uses navigation menu
   - Clicks logo to return home
   - Uses browser back button

TECHNICAL FLOWS:
---------------

FLOW 1: DYNAMIC PRODUCT LOADING
--------------------------------
1. Page loads (index.php or products.php)
2. JavaScript executes on DOMContentLoaded
3. Fetch request to get_products.php
4. Server queries products table
5. Returns JSON data
6. JavaScript creates product cards
7. Cards inserted into DOM
8. User sees products

FLOW 2: CART OPERATIONS
------------------------
1. User clicks add to cart
2. JavaScript collects product data
3. FormData created with product info
4. Fetch POST to add_to_cart.php
5. Server inserts into cart table
6. Returns JSON success
7. JavaScript redirects to cart.php
8. Cart page queries cart table
9. Displays cart items

FLOW 3: ORDER PROCESSING
-------------------------
1. User clicks "Buy Now" on checkout
2. Form submits to process_order.php
3. Server:
   - Generates unique order number
   - Creates order record
   - Creates order_items records
   - Calculates totals
   - Updates database
4. Redirects to order_success.php
5. User sees confirmation

FLOW 4: SEARCH FUNCTIONALITY
------------------------------
1. User types in search bar
2. Presses Enter key
3. JavaScript captures query
4. Redirects to search_results.php?query=...
5. Server processes query
6. Searches products table
7. Returns matching products
8. Displays results

ERROR HANDLING FLOWS:
--------------------

FLOW 1: DATABASE CONNECTION FAILURE
-----------------------------------
- Connection error message displayed
- Page may continue with limited functionality
- Error logged

FLOW 2: CART OPERATION FAILURE
-------------------------------
- Error message shown to user
- Cart state preserved
- User can retry operation

FLOW 3: ORDER PROCESSING FAILURE
---------------------------------
- Error message displayed
- Order not created
- User returned to checkout
- Can retry purchase

FLOW 4: INVALID USER INPUT
--------------------------
- Form validation errors shown
- User prompted to correct input
- Submission prevented until valid
